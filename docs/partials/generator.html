Calculated panel reactive antibodies

Each row represents one band. Use the format 0.2 0.11 to represent a band with probability 0.2 in which all patients have cRF 0.11. Use the format 0.2 0.4 0.5 to represent a band with probability 0.2 in which cRF values are uniformly distributed on the interval from 0.4 to 0.5.

Recipients who have a blood-group compatible donor should have the same different cPRA distributions to recipients who don’t have a blood-group compatible donor.

Compatibility calculations

This field allows the user to specify the exact relationship between cPRA and chance of compatibility. Each row represents one band, or set of cPRA values, as follows. l h c m. Such an entry is to be read as "If a recipient’s cPRA falls in the range [l, h), then their chance of compatibility is 1 - c - m*cPRA/100. As a comparison, the entry 0 101 0 1 means that any recipient whose cPRA is in the range [0, 101) (i.e., all recipients) have a chance of compatibility of 1 - cPRA/100.

Tuning

Generators will by default discard compatible pairs. This is done to attempt to simulate the effect that occurs as compatible pairs simply arrange transplants directly and avoid kidney exchange programmes altogether. However, this means that input characteristics (e.g., number of donors per recipient) won’t necessarily match output characteristics (e.g., recipients with a higher number of paired donors are more likely to have a compatible donor, and therefore will be underrepresented in the ouptut). To avoid this, tuning creates random instances, compares various input and output distributions, and then adapts the actual input distributions to closely match the target distributions. This process can be repeated for a number of iterations, or until any error is sufficiently small.

Other options

The number of patients per instance must be in range 10 to 10000.
The number of instances must be at least 1 and at most 20.
The altruistic proportion must be at least 0 and at most 0.5.
